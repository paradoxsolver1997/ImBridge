Image Formats: Bitmap vs Vector

This document explains the key differences between bitmap and vector image formats, and clarifies how parameters like pixel size, DPI, and page size affect their use in display and printing workflows.

Bitmap (Raster) Images

  • Formats: PNG, JPEG, BMP, TIFF, GIF, etc.
  • Core property: Pixel dimensions (width Ã— height)
  • DPI (Dots Per Inch):
    • Stored as metadata in some formats (e.g., PNG, TIFF, JPEG)
    • DPI only matters for printing or when embedding into vector/page formats
    • For display on screens, only pixel size is used; DPI is ignored
  • Physical size calculation:
    • Physical width (inches) = pixel width / DPI
    • Physical height (inches) = pixel height / DPI
  • Common confusion:
    • Changing DPI does not change pixel data, only how large the image prints
    • For web/screen, DPI is irrelevant

Vector Images

  • Formats: SVG, PDF, EPS, PS, AI, etc.
  • Core property: Geometric shapes, paths, and text
  • Page size:
    • Vector formats define a page or canvas size in physical units (pt, mm, in)
    • Example: PDF/EPS/PS use pt (1pt = 1/72 inch); SVG can use px, mm, in, pt
  • Embedded bitmaps:
    • Vector files can embed bitmap images
    • The physical size of embedded bitmaps is determined by their pixel size and assigned DPI
  • DPI in vector formats:
    • SVG: No global DPI; size depends on units (px, mm, in, pt)
    • PDF/EPS/PS: Page size is physical; embedded bitmaps use DPI to map pixels to physical size

Display vs Print

  • Display (screen):
    • Bitmap: Pixel size is all that matters
    • Vector: Rendered to screen pixels; page size and DPI are ignored
  • Print:
    • Bitmap: DPI determines printed size
    • Vector: Page size determines printed area; embedded bitmaps use DPI for scaling

Practical Tips

  • Bitmap DPI:
    • Set DPI only if you care about print size
    • For screen use, ignore DPI
  • Vector page size:
    • Always check units (pt, mm, in, px)
    • For precise print output, use physical units
  • SVG:
    • Use width/height with units for physical size
    • Use viewBox for scalable graphics
  • PDF/EPS/PS:
    • Page size is always physical
    • Embedded bitmaps: DPI controls print size

Common User Confusions

  • "Why does my image print so small/large?"
    • Check DPI and pixel size for bitmaps
    • Check page size and units for vector files
  • "Why does DPI not affect my image on screen?"
    • DPI is only for print or embedding, not for display
  • "How do I control SVG print size?"
    • Use width/height with units (e.g., mm, in, pt)
